Visual Language & Product Identity

The visual philosophy behind every screen, chart, and interaction

01 Brand Personality

How Advizmo should feel — and how it should never feel.

How Advizmo Should Feel

🧘
Calm
Reduces financial anxiety through soft colors, generous whitespace, gentle animations
💪
Confident
Communicates competence through clear hierarchy, strong typography, purposeful design
🤝
Trustworthy
Builds financial trust through consistent patterns, transparent information, reliable feedback
🧠
Intelligent
Shows smart automation through clean data visualization, insightful defaults, AI transparency
🔍
Transparent
Never hides information through clear labels, honest states, upfront costs
🤗
Helpful
Guides without overwhelming through progressive disclosure, contextual help, sensible defaults
👔
Professional
Feels like a premium tool through refined details, polished interactions, attention to quality

How Advizmo Should Never Feel

✕ Never Anxious

Cluttered, noisy, urgent designs increase financial stress

✕ Never Playful

Cartoonish, bouncy, casual designs undermine financial trust

✕ Never Aggressive

Flashing, demanding, loud designs feel like a casino

✕ Never Cold

Robotic, impersonal, sterile designs lack human connection

02 Visual Principles

The core visual principles that guide every design decision.

Less UI, More Information

Reduce chrome and interface elements. Let content speak.

Less Decoration, More Clarity

Remove anything that doesn't aid comprehension.

Less Color, More Contrast

Use color sparingly. Contrast creates hierarchy.

Less Complexity, More Confidence

Simple interfaces feel more trustworthy.

Less Noise, More Breathing Space

Whitespace is a feature, not wasted space.

Good vs Bad Examples

✓ Clean Data Table

Minimal borders, clear hierarchy, generous whitespace

✕ Heavy Data Table

Heavy borders, background colors, decorative elements

✓ Simple Chart

Clear labels, minimal decoration, purposeful color

✕ Decorative Chart

Gradient backgrounds, shadows, decorative icons

07 Color Philosophy

How color should be used — and how it should never be used.

Blue
Trust and Primary Action
Primary buttons, links, focus states
Green
Financial Growth and Success
Gains, positive changes, success states
Orange
Attention
Warnings, pending states, attention-required
Red
Risk and Critical Issues
Losses, errors, destructive actions
Purple
AI and Intelligence
AI insights, smart suggestions, automation

Color Rules

✓
80% neutral colors — The interface should be dominated by neutral colors
✓
Brand colors guide attention — Use brand colors sparingly and purposefully
✓
Color for status, not decoration — Use color to communicate status, not for decoration
✓
Never use color alone — Always pair color with text or icons for accessibility

Typography Rules

✓
Use size and weight for hierarchy — Not color, not decoration

Tabular numerals for data — Use tabular numerals for financial data

Don't rely on bold everywhere — Use bold sparingly for emphasis
✓
Consistent line height — 1.5 for body, 1.2 for headings

14 AI Personality

How AI features should feel — and how they should never feel.

📚
Knowledge
Show confidence levels, explain reasoning, provide context
🔍
Transparency
Show how AI works, what data it uses, what confidence it has
🤝
Assistance
Position AI as a tool, not a mystery
💪
Confidence
Show uncertainty when appropriate, don't overstate confidence

AI Rules

✓ Transparent AI

"Based on your spending patterns, we recommend..." (85% confidence)

✕ Mysterious AI

"Our magic algorithm knows what you need!"

15 Financial Personality

How financial information should feel — and how it should never feel.

✅
Reliable
Consistent formatting, clear labels, accurate data
🏛️
Stable
Avoid sudden changes, provide context, show trends
📋
Structured
Clear hierarchy, consistent patterns, logical organization
💡
Easy to Understand
Plain language, provide context, avoid jargon

Financial Rules

✓ Calm Financial

Clean portfolio view, clear numbers, gentle trends

✕ Casino-like

Flashy animations, bright colors, gambling metaphors

Whitespace Rules

✓
Whitespace signals importance — More whitespace = more important
✓
Whitespace groups elements — Use whitespace to show relationships
✓
Whitespace creates focus — Use whitespace to isolate important elements
✓
Every whitespace decision should have a reason — Never use arbitrary spacing
