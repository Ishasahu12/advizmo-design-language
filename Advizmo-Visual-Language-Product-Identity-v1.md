## 1. Brand Personality

### How Advizmo Should Feel

| Trait | Description | Visual Expression |
|-------|-------------|-------------------|
| **Calm** | Reduces financial anxiety | Soft colors, generous whitespace, gentle animations |
| **Confident** | Communicates competence | Clear hierarchy, strong typography, purposeful design |
| **Trustworthy** | Builds financial trust | Consistent patterns, transparent information, reliable feedback |
| **Intelligent** | Shows smart automation | Clean data visualization, insightful defaults, AI transparency |
| **Transparent** | Never hides information | Clear labels, honest states, upfront costs |
| **Helpful** | Guides without overwhelming | Progressive disclosure, contextual help, sensible defaults |
| **Professional** | Feels like a premium tool | Refined details, polished interactions, attention to quality |

### How Advizmo Should Never Feel

| Anti-Trait | What It Looks Like | Why It's Wrong |
|------------|-------------------|----------------|
| **Anxious** | Cluttered, noisy, urgent | Increases financial stress |
| **Playful** | Cartoonish, bouncy, casual | Undermines financial trust |
| **Aggressive** | Flashing, demanding, loud | Feels like a casino |
| **Cold** | Robotic, impersonal, sterile | Lacks human connection |
| **Trendy** | Following fashions, dated quickly | Won't age well |
| **Decorative** | Ornamental, unnecessary visuals | Distracts from information |

## 2. Visual Principles

### Less UI, More Information
Reduce chrome and interface elements. Let content speak.

**Good**: Clean data table with minimal borders
**Bad**: Heavy borders, background colors, decorative elements

### Less Decoration, More Clarity
Remove anything that doesn't aid comprehension.

**Good**: Simple chart with clear labels
**Bad**: Gradient backgrounds, shadows, decorative icons

### Less Color, More Contrast
Use color sparingly. Contrast creates hierarchy.

**Good**: Black text on white background, blue for actions
**Bad**: Rainbow of colors, gradient text, colored backgrounds

### Less Complexity, More Confidence
Simple interfaces feel more trustworthy.

**Good**: Clear single-column form
**Bad**: Dense multi-column form with many options

### Less Noise, More Breathing Space
Whitespace is a feature, not wasted space.

**Good**: Generous padding, clear sections
**Bad**: Cramped layouts, minimal margins

## 3. Information Hierarchy

### Primary Information
The most important data on any screen.

**Examples**: Portfolio balance, account balance, total net worth
**Visual Treatment**: Largest typography, highest contrast, prominent placement

### Secondary Information
Supporting data that adds context.

**Examples**: Daily change, monthly performance, transaction count
**Visual Treatment**: Medium typography, good contrast, adjacent to primary

### Supporting Information
Additional details that help understanding.

**Examples**: Account type, last updated, transaction category
**Visual Treatment**: Smaller typography, lower contrast, accessible but not dominant

### Metadata
Technical information for reference.

**Examples**: Account numbers, timestamps, system IDs
**Visual Treatment**: Smallest typography, lowest contrast, available but hidden

### Background Information
Contextual information that rarely needs attention.

**Examples**: Terms, conditions, legal text
**Visual Treatment**: Muted, collapsible, only visible when requested

---

## 4. Visual Hierarchy

### Scale
Larger elements draw attention first.

**Rule**: Primary information should be 2–3x larger than supporting information.

### Contrast
High contrast elements stand out.

**Rule**: Primary actions should have 7:1 contrast ratio minimum.

### Position
Top-left (LTR) elements are seen first.

**Rule**: Place primary information at the top of cards and screens.

### Whitespace
Surrounded elements draw attention.

**Rule**: Isolate important elements with generous whitespace.

### Typography
Bold, large text draws attention.

**Rule**: Use weight and size, not color, for hierarchy.

### Elevation
Higher elevation elements appear more important.

**Rule**: Use elevation sparingly — only for modals and floating elements.

### Color
Colored elements draw attention.

**Rule**: Use color for status and action, not decoration.

### Motion
Moving elements draw attention.

**Rule**: Use motion for feedback, not decoration.

## 5. Layout Rhythm

### Vertical Rhythm
Consistent vertical spacing between elements.

**Rule**: Use the 4pt grid for all vertical spacing.

### Horizontal Rhythm
Consistent horizontal spacing between elements.

**Rule**: Use the 4pt grid for all horizontal spacing.

### Component Rhythm
Consistent spacing within components.

**Rule**: Use 16px padding for cards, 12px for buttons, 8px for chips.

### Section Rhythm
Consistent spacing between sections.

**Rule**: Use 24px between sections, 32px between major sections.

### Dashboard Rhythm
Consistent spacing across dashboard layouts.

**Rule**: Use 16px grid gaps, 24px section gaps.

### Reading Rhythm
Consistent line height and paragraph spacing.

**Rule**: Use 1.5 line height for body text, 1.2 for headings.

---

## 6. Density

### Compact Density
Maximum information in minimum space.

**When to use**: Data tables, lists with many items, power user views
**Spacing**: 8px padding, 4px gaps
**Typography**: 12–14px

### Comfortable Density
Balanced information and whitespace.

**When to use**: Default for most screens, forms, cards
**Spacing**: 16px padding, 8px gaps
**Typography**: 14–16px

### Relaxed Density
Maximum whitespace, minimal information.

**When to use**: Onboarding, empty states, important announcements
**Spacing**: 24px padding, 16px gaps
**Typography**: 16–18px

## 7. Color Philosophy

### Blue — Trust and Primary Action
Blue represents trust, reliability, and primary actions.

**Use for**: Primary buttons, links, focus states, brand elements
**Don't use for**: Backgrounds, decorative elements, non-interactive elements

### Green — Financial Growth and Success
Green represents positive financial outcomes.

**Use for**: Gains, positive changes, success states, connected accounts
**Don't use for**: General actions, decorative elements, non-financial success

### Orange — Attention
Orange draws attention to important information.

**Use for**: Warnings, pending states, attention-required items
**Don't use for**: Primary actions, backgrounds, decorative elements

### Red — Risk and Critical Issues
Red represents risk, errors, and critical information.

**Use for**: Losses, errors, destructive actions, critical alerts
**Don't use for**: Warnings, non-critical attention, decorative elements

### Purple — AI and Intelligence
Purple represents AI features and intelligent automation.

**Use for**: AI insights, smart suggestions, automation indicators
**Don't use for**: General actions, backgrounds, non-AI features

### Neutral Colors — Interface Foundation
Neutral colors dominate the interface.

**Use for**: Backgrounds, text, borders, subtle elements
**Rule**: 80% of the interface should be neutral colors

### Brand Colors — Guided Attention
Brand colors guide attention, not overwhelm.

**Use for**: Primary actions, important information, status indicators
**Rule**: Brand colors should be used sparingly and purposefully

## 8. Typography Philosophy

### Typography Hierarchy
Clear hierarchy guides the eye.

**Rule**: Use size and weight, not color, for hierarchy.

### Reading Rhythm
Consistent line height and spacing.

**Rule**: 1.5 line height for body, 1.2 for headings.

### Scanning Behaviour
Users scan, don't read.

**Rule**: Use headings, labels, and whitespace to enable scanning.

### Numeric Typography
Numbers need special treatment.

**Rule**: Use tabular numerals for data, proportional for text.

### Financial Typography
Financial data needs clarity.

**Rule**: Use clear, unambiguous fonts for numbers.

### Large Numbers
Large numbers need careful treatment.

**Rule**: Use appropriate size, weight, and spacing for large numbers.

### Data Tables
Data tables need readable typography.

**Rule**: Use consistent alignment, spacing, and weight.

### Charts
Chart typography needs to be clear but not dominant.

**Rule**: Use smaller, lighter typography for chart labels.

### Emphasis
Emphasis should be purposeful.

**Rule**: Don't rely on bold everywhere — use it sparingly.

## 9. Shape Language

### Rounded Corners
Soft, approachable, modern.

**Rule**: Use 8px radius for small elements, 12px for cards, 9999px for pills.

### Containers
Contain content without visual weight.

**Rule**: Use subtle borders or background color, not heavy shadows.

### Cards
Present information in digestible chunks.

**Rule**: Use consistent padding, clear hierarchy, subtle elevation.

### Buttons
Communicate actions clearly.

**Rule**: Use consistent sizing, clear labels, obvious states.

### Fields
Collect input clearly.

**Rule**: Use consistent padding, clear labels, obvious states.

### Chips
Show selectable options compactly.

**Rule**: Use consistent sizing, clear labels, obvious states.

### Tags
Show categories and labels.

**Rule**: Use consistent sizing, clear labels, subtle styling.

### Charts
Present data clearly.

**Rule**: Use consistent styling, clear labels, appropriate density.

## 10. Iconography

### Outline Icons
Clean, modern, consistent.

**Rule**: Use outline style, not filled.

### 2px Stroke
Consistent stroke width.

**Rule**: Use 2px stroke for all icons.

### Optical Alignment
Icons should feel aligned, not mathematically aligned.

**Rule**: Adjust icon position for visual balance.

### Visual Balance
Icons should feel balanced.

**Rule**: Ensure consistent visual weight across icons.

### Corner Treatment
Icons should have consistent corner treatment.

**Rule**: Use rounded corners, match component radius.

### Grid
Icons should be designed on a grid.

**Rule**: Use 24x24 grid for all icons.

### Style Consistency
Never mix icon styles.

**Rule**: Use the same style, weight, and treatment for all icons.

## 11. Imagery

### Photography
Real, authentic, relevant.

**Rule**: Use authentic photography, not generic stock.

### People
Real people, diverse, relatable.

**Rule**: Show diverse people in realistic settings.

### Devices
Show the product in context.

**Rule**: Show the product on real devices in real environments.

### Lifestyle
Show the product's role in daily life.

**Rule**: Show the product fitting naturally into daily routines.

### Macro
Show details when relevant.

**Rule**: Use macro photography for important details.

### Backgrounds
Subtle, non-distracting.

**Rule**: Use subtle backgrounds that don't compete with content.

### Avoid Generic Stock Photos
Generic photos reduce trust.

**Rule**: Never use obviously staged or generic stock photography.

## 12. Illustration

### Illustration Style
Clean, geometric, purposeful.

**Rule**: Use clean geometric illustrations, not hand-drawn.

### Geometry
Simple geometric shapes.

**Rule**: Use basic geometric shapes, not complex forms.

### Perspective
Flat or simple isometric.

**Rule**: Use flat or simple isometric perspective.

### Color
Limited, purposeful color palette.

**Rule**: Use the same color palette as the product.

### Lighting
Soft, even lighting.

**Rule**: Use soft, even lighting, not dramatic shadows.

### Financial Metaphors
Illustrate financial concepts clearly.

**Rule**: Use metaphors that reinforce financial understanding.

### Don't Use Childish Illustrations
Childish illustrations undermine trust.

**Rule**: Never use cartoonish or childish illustration styles.

## 13. Charts Personality

### Calm
Charts should feel calm, not exciting.

**Rule**: Use gentle colors, smooth animations, clear labels.

### Precise
Charts should feel precise and accurate.

**Rule**: Use exact values, clear scales, accurate representations.

### Readable
Charts should be easy to read.

**Rule**: Use appropriate size, clear labels, sufficient contrast.

### Analytical
Charts should support analysis.

**Rule**: Enable interaction, show details, support exploration.

### Never Playful
Playful charts undermine trust.

**Rule**: Never use bouncy animations, bright colors, or casual styling.

### Never Decorative
Decorative charts distract from data.

**Rule**: Never add unnecessary elements that don't aid understanding.

---

## 14. AI Personality

### Knowledge
AI should communicate knowledge.

**Rule**: Show confidence levels, explain reasoning, provide context.

### Transparency
AI should be transparent.

**Rule**: Show how AI works, what data it uses, what confidence it has.

### Assistance
AI should feel helpful, not magical.

**Rule**: Position AI as a tool, not a mystery.

### Confidence
AI should communicate confidence appropriately.

**Rule**: Show uncertainty when appropriate, don't overstate confidence.

### Never Mystery
Mysterious AI reduces trust.

**Rule**: Never hide how AI works or what it's doing.

### Never Magic
Magic-like AI reduces trust.

**Rule**: Never present AI as magical or all-knowing.

---

## 15. Financial Personality

### Reliable
Financial information should feel reliable.

**Rule**: Use consistent formatting, clear labels, accurate data.

### Stable
Financial information should feel stable.

**Rule**: Avoid sudden changes, provide context, show trends.

### Structured
Financial information should feel structured.

**Rule**: Use clear hierarchy, consistent patterns, logical organization.

### Easy to Understand
Financial information should be easy to understand.

**Rule**: Use plain language, provide context, avoid jargon.

### Never Casino-like
Casino-like design undermines trust.

**Rule**: Never use flashy animations, bright colors, or gambling metaphors.

### Never Trading-platform Aggressive
Aggressive trading platforms create anxiety.

**Rule**: Never use urgent language, countdown timers, or pressure tactics.

## 16. Empty Space

### Importance
Whitespace signals importance.

**Rule**: More whitespace = more important.

### Grouping
Whitespace groups related elements.

**Rule**: Use whitespace to show relationships.

### Hierarchy
Whitespace creates hierarchy.

**Rule**: Use whitespace to guide the eye.

### Focus
Whitespace creates focus.

**Rule**: Use whitespace to isolate important elements.

### Calmness
Whitespace creates calm.

**Rule**: Use whitespace to reduce visual noise.

### Intentional Usage
Whitespace should be intentional.

**Rule**: Every whitespace decision should have a reason.
