Typography Rules
One font family only: Inter
Three weights only: 400 Regular, 600 Semibold, 700 Bold
Never use: Italic, 300 Light, 800-900 Extra Bold
Alignment: Left-align all text
Center only: Buttons, badges, empty states
Line length: Max 80 characters
Line height: 1.2× to 1.4× font size

Borders

Borders define structure. Use sparingly — whitespace communicates separation better.

Border Scale
TokenWidthUsage
border-default1pxDefault borders, dividers
border-strong2pxEmphasis, section breaks
border-focus2pxFocus rings (offset 2px)

Border Rules
Prefer no border. Use space first.
If border needed, use 1px default.
2px only for emphasis or focus.
Borders should not compete with content.

08

Iconography

Outline icons only. Material Symbols Rounded. Consistent with iOS. Clean and readable at all sizes.

Icon Sizes
SizeUsageContext
16pxCompactBadges, inline icons
20pxSmallList item icons
24pxStandardNavigation, actions
28pxLargeEmpty state icons
32pxXLHero icons
Icon Rules
Outline only. Never mix with filled.
2px stroke only. No 1.5px or 2.5px.
Round corners only. No sharp corners.
SF Symbols preferred. Use custom only when SF Symbols lacks an icon.
Test at 16px and 24px. If unreadable, redesign.
Never use icons alone. Always pair with text or clear context.

Naming Rules

Be consistent. Same pattern everywhere.
Be predictable. Name matches function.

Component Rules

Minimum components. Maximum reuse. Every component must justify its existence.

Component Categories
CategoryCountExamples
Actions1Button
Inputs1Input
Containers1Card
Navigation2NavigationBar, TabBar
Feedback3Alert, Badge, Toast
Display2Avatar, Divider
Data2ListItem, Progress
Overlay2Sheet, Dialog
Rules for All Components
Padding: Use spacing constants (space-2, space-4, etc.)
Colors: Use semantic tokens only
Typography: Use typography tokens
Radius: Follow radius rules
States: Define all states (default, hover, pressed, disabled, loading)
Accessibility: All interactive elements have labels
Touch target: Minimum 44×44pt
