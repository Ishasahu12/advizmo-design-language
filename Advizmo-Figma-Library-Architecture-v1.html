Figma Library Architecture

The official architecture for every future Advizmo design file

01 File Philosophy

The foundational principles that guide every decision in the library.

Single Source of Truth

One component exists in one place. Every instance traces back to a single master.

Composition over Duplication

Build complex components from simpler ones. A Bill Card is a Card + Avatar + Typography.

Variants over Copies

Never duplicate a component. Use Figma's variant system for states, sizes, and types.

Variables over Styles

Use Figma Variables for all values that change across themes, sizes, or contexts.

Auto Layout Everywhere

Every published component must use Auto Layout. No exceptions.

Documentation First

Every component must be documented before it is published.

Engineering Friendly

Components must map to engineering reality. Property names should match code.

Scalable Naming

Names must work at scale. Use systematic, predictable naming.

✓
Variable Driven — Uses tokens, not hard-coded values
✓

✓
Documented — Purpose, usage, examples
✓
Engineering Ready — Maps to code
✓
