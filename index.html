Typography Rules
One font family: Inter for UI, IBM Plex Sans for financial values only
Three weights: 400 Regular, 600 Semibold, 700 Bold (500 Medium for labels)
Never use: Italic, 300 Light, 800-900 Extra Bold
Left-align all text. Center only: buttons, badges, empty states
Max line length: 80 characters
Line height: 1.2× to 1.4× font size
Financial values: always use tabular nums (font-feature-settings: tnum)

Icon Rules
Outline only. Never mix with filled variants.
2px stroke. No 1.5px or 2.5px.
Rounded corners only. No sharp corners.
Optically aligned within 24×24 bounding box.
Test at 16px and 24px. If unreadable, redesign.
Never use icons alone. Always pair with text or clear context.
Never use filled icons. Outline only for consistency.
Never use icons smaller than 16px.

19 · Token Architecture3-tier hierarchy
Tier 1 — Primitives
WhatRaw values. Named by value. No context.
Exampleblue.600, spacing.4, radius.sm
Fileprimitives.json
Change frequencyRarely. Brand refresh only.
Tier 2 — Semantic
WhatNamed by purpose. Maps to primitives.
Exampleaction.primary → blue.600, text.primary → gray.900
Filesemantic-light.json · semantic-dark.json
Change frequencyModerately. Theme updates.
Tier 3 — Component
WhatComponent-specific. Maps to semantics.
Examplebutton.primary.bg → action.primary
Filecomponents.json
Change frequencyOften. Component iterations.
Flow
Primitives → Semantic → Component → UI

Naming Convention
ComponentPascalCase: NetWorthCard, PortfolioChart
PropcamelCase: variant, size, disabled
Tokendot.notation: action.primary, text.secondary
Filekebab-case: net-worth-card.tsx
Component API Rules
Every component must accept: variant, size, disabled, loading.
Financial components must accept: currency, locale, tabular.
All components must support: dark mode, reduced motion, RTL.
Never hardcode values. Always use tokens.
One component, one file. No mega-components.
Props over config objects. Explicit is better than implicit.
