> Scope: Complete audit and repair of typography visibility in Light Mode

Executive Summary

Problem: Typography appears almost invisible in Light Mode. Root Cause: Typography styles may be incorrectly bound to:
  • Dark mode color variables in Light Mode
  • Low-contrast color tokens
  • Reduced opacity values
  • Hardcoded colors instead of semantic variables
    • Solution: This document provides:
  • Complete audit of all semantic text tokens
  • Correct Light Mode and Dark Mode values
  • Step-by-step fix instructions
  • Verification checklist

    3. Common Issues & Fixes

    3.1 Issue: Typography Using Dark Mode Values in Light Mode

    Symptom: Text appears very light/invisible on white background Cause: Typography styles bound to Dark Mode variables: 
    ❌ INCORRECT:

    text-primary → gray-50 (#F8FAFC)  // This is Dark Mode value!
    

    text-secondary → gray-400 (#94A3B8)  // This is Dark Mode value!
    Fix: Rebind to Light Mode values: 
    ✅ CORRECT:

    text-primary → gray-900 (#111827)  // Light Mode value
    

    text-secondary → gray-600 (#4B5563)  // Light Mode value

    3.2 Issue: Typography Using Low-Contrast Tokens

    Symptom: Text is visible but hard to read Cause: Using tertiary or disabled tokens for primary content: 
    ❌ INCORRECT:

    Heading → text-tertiary (#6B7280)
    

    Body → text-disabled (#9CA3AF)
    Fix: Use appropriate hierarchy tokens: 
    ✅ CORRECT:

    Heading → text-primary (#111827)
    

    Body → text-primary (#111827)
    

    Caption → text-secondary (#4B5563)

    3.3 Issue: Typography with Reduced Opacity

    Symptom: Text appears faded or washed out Cause: Opacity applied to text layer: 
    ❌ INCORRECT:

    Layer opacity: 50%
    

    Layer opacity: 70%
    Fix: Remove opacity, use color token instead: 
    ✅ CORRECT:

    Layer opacity: 100%
    

    Color: text-secondary (for lighter text)

    3.4 Issue: Typography Using White on Light Background

    Symptom: Text completely invisible on white Cause: text-inverse or text-on-color used on light background: 
    ❌ INCORRECT:

    Card background: white
    

    Text color: text-inverse (white)  // White on white!
    Fix: Use appropriate text token: 
    ✅ CORRECT:

    Card background: white
    

    Text color: text-primary (#111827)

    3.5 Issue: Typography Using Hardcoded Colors

    Symptom: Colors don't update with theme changes Cause: Hex values instead of variables: 
    ❌ INCORRECT:

    Color: #FFFFFF
    

    Color: #000000
    

    Color: #666666
    Fix: Use semantic tokens: 
    ✅ CORRECT:

    Color: text-primary
    

    Color: text-secondary
    

    Color: text-tertiary

    4. Figma Repair Instructions

    4.1 Step 1: Identify Affected Layers

  • Open Figma file
  • Select all text layers (Cmd/Ctrl + A, then filter by Text)
  • Inspect each layer's fill color
  • Check if color is a variable or hardcoded hex

    • 4.2 Step 2: Check Variable Bindings

    For each text layer:

  • Select the layer
  • Look at Fill section in right panel
  • Check if it shows a variable name (e.g., text/primary)
  • If it shows hex value, it's not using variables

    • 4.3 Step 3: Fix Incorrect Bindings

    For each text layer with incorrect binding:

  • Select the layer
  • Click on the color fill
  • Click "Local styles" or "Variables" tab
  • Select correct semantic token:

    • - Headings → text/primary

    - Body text → text/primary

    - Captions → text/secondary

    - Placeholders → text/tertiary

    - Disabled → text/disabled

    4.4 Step 4: Remove Opacity

    For each text layer with opacity < 100%:

  • Select the layer
  • Find "Layer" section in right panel
  • Set opacity to 100%
  • If text needs to be lighter, use text-secondary or text-tertiary instead

    • 4.5 Step 5: Verify Light Mode

  • Switch to Light Mode in Figma
  • Scroll through all pages
  • Verify all text is readable
  • Check contrast ratios using Stark plugin or manual calculation

    10. Verification Steps

    10.1 Visual Verification

  • Open Figma file
  • Switch to Light Mode
  • Navigate to Typography section
  • Verify all text is readable
  • Check each style:

    • - Display: Dark text on light bg

    - Headings: Dark text on light bg

    - Body: Dark text on light bg

    - Captions: Medium text on light bg

    - Placeholders: Light text on light bg

    10.2 Technical Verification

  • Select each text layer
  • Check Fill section:

    • - Should show variable name (e.g., text/primary)

    - Should NOT show hex value

    - Should NOT show opacity < 100%

  • Check if variable resolves to correct hex:

    • - text/primary → #111827 (Light) or #F8FAFC (Dark)

    10.3 Accessibility Verification

  • Use Stark plugin to check contrast
  • Or manually calculate:

    • - Contrast ratio = (L1 + 0.05) / (L2 + 0.05)

    - L1 = relative luminance of lighter color

    - L2 = relative luminance of darker color

  • Verify all text meets WCAG AA (4.5:1 for normal text)

    • 11. Troubleshooting

    Issue: Text still invisible after fixing variables

    Check:
  • Is the text layer inside a frame with opacity?
  • Is the text layer's blend mode correct?
  • Is there a dark overlay on top?
  • Is the parent frame's background correct?

    • Issue: Text colors don't change with theme

    Check:
  • Are variables bound to the correct collection?
  • Is the mode correctly set?
  • Are there any overrides on the text layer?

    • Issue: Some text is readable, some is not

    Check:
  • Different text layers may have different bindings
  • Check each layer individually
  • Look for layers that might be using local styles instead of variables
