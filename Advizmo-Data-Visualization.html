7.1 · Visualization Principles
The foundation of every chart
Core Philosophy
Every visualization must answer one question. Never combine multiple stories into one chart.
Visual hierarchy: Title → Primary KPI → Visualization → Supporting Insight → Legend → Timestamp
Every chart must communicate: Trust, Simplicity, Financial clarity, Precision, Calm.
Never decoration. Every pixel earns its place.
Good
How is my portfolio performing?
One chart, one story
Clean grid lines
Clear delta values
Timestamp always visible
Bad
Portfolio + Tax + Bills + Spending + Goals
Multiple unrelated datasets
Cluttered grid lines
Missing context
No indication of data freshness

7.12 · Accessibility
Every chart must support all users
Requirements
record_voice_overVoiceOver
Every chart must have a screen reader description summarizing the key insight.
contrastHigh Contrast
All chart elements must meet WCAG AA 4.5:1 contrast ratio minimum.
paletteColor-Blind Safe
Never rely on color alone. Use patterns, labels, and icons alongside.
format_sizeDynamic Type
All text in charts must scale with iOS Dynamic Type settings.
animationReduce Motion
Respect system reduce motion. Replace animations with static state changes.
keyboardKeyboard Nav
Web charts must be fully navigable via keyboard with visible focus states.
Minimum Touch Target
44×44pt — Every interactive chart element must meet this minimum.
