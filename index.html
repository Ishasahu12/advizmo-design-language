Design Principles

1. Reduce Financial Anxiety

Financial applications cause stress. Design should use calming motion, present information in digestible chunks, and provide clear feedback for all actions.

2. Automate Complexity

Show automated states clearly, explain what's happening without requiring user action, and make complex features discoverable but not overwhelming.

3. Clarity Before Decoration

Every visual element must earn its place. Remove decorative elements that don't aid comprehension.

4. Consistency Creates Trust

Users should be able to predict where things will be and expect similar elements to behave similarly.

5. Design for Daily Use

Optimize for repeated interactions with comfortable touch targets and visible state changes.

6. Highlight What Matters

Primary actions are visually dominant. Secondary actions are present but subdued.

7. Reduce Cognitive Load

Don't make users think. Use familiar patterns, name things clearly, and provide sensible defaults.

8. Engineering Simplicity

Design should be implementable. Avoid complex conditional styling and prefer composition over configuration.
