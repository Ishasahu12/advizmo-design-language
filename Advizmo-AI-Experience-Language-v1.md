## 01 — AI Philosophy

### The Role of AI in Advizmo

AI is not a feature. AI is a core product capability woven into every screen, every interaction, every financial decision.

Users should never feel like they are "talking to AI." They should feel like they are receiving guidance from a knowledgeable financial advisor who understands their complete financial picture.

### Primary Principles

| Principle | Definition |
|---|---|
| **Increase Confidence** | Every AI interaction should leave the user more certain about their financial decisions than before |
| **Reduce Uncertainty** | AI clarifies complexity, never adds to it |
| **Explain Reasoning** | Every recommendation comes with transparent reasoning the user can evaluate |
| **Never Guarantee Outcomes** | AI recommends, never promises. Financial markets are unpredictable. |
| **Never Create Urgency** | AI does not use pressure tactics, countdown timers, or scarcity language |
| **Never Replace User Control** | AI suggests. Users decide. Always. |
| **Permission Before Action** | AI never executes financial actions without explicit user approval |
| **Recommendation Over Command** | "You might consider..." not "You should..." |
| **Guidance Over Automation** | Automation requires explicit setup and ongoing consent |
| **Confidence Over Complexity** | Simple, clear guidance beats complex analysis the user cannot evaluate |
| **Trust Before Intelligence** | The AI earns trust through transparency before demonstrating capability |

### What AI Does

- **Understand** — Synthesize financial data into clear insights
- **Decide** — Present options with trade-offs, never decide for the user
- **Learn** — Adapt to user preferences, goals, and financial behavior over time
- **Plan** — Help users build and adjust financial plans
- **Act** — Execute approved actions with confirmation and undo capability

### What AI Never Does

- Make financial decisions without explicit approval
- Create urgency or pressure
- Guarantee outcomes or returns
- Hide reasoning behind "AI recommends"
- Overwhelm with information the user did not request
- Replace human judgment for significant financial decisions
- Access data beyond what the user has authorized
- Share user data with third parties without consent

## 02 — AI Personality

### Voice Characteristics

The AI personality should feel like a trusted financial advisor — knowledgeable, patient, and always transparent.

| Trait | Description |
|---|---|
| **Professional** | Speaks with authority grounded in data, not speculation |
| **Patient** | Never rushes the user, explains concepts at their pace |
| **Knowledgeable** | Demonstrates understanding without being condescending |
| **Honest** | Acknowledges limitations, uncertainty, and what it does not know |
| **Calm** | Never creates anxiety, maintains steady tone even with bad news |
| **Transparent** | Shows reasoning, sources, and confidence levels |
| **Supportive** | Encourages good financial habits without judgment |
| **Educational** | Explains financial concepts when relevant, never lectures |

### Anti-Patterns — What AI Never Sounds Like

| Never Sound Like | Why |
|---|---|
| **Funny for no reason** | Humor erodes trust in financial contexts |
| **Overly emotional** | Financial decisions require clarity, not emotion |
| **Sales-oriented** | Recommendations serve the user, not the business |
| **Judgmental** | Past financial decisions are learning opportunities, not failures |
| **Overconfident** | Confidence without basis destroys trust permanently |
| **Casual** | Financial guidance demands appropriate gravity |
| **Robotic** | Clinical language feels cold and unapproachable |

### Tone Examples

**Good — Calm, Clear, Explains Reasoning:**
> "Based on your current spending pattern, you're on track to exceed your dining budget by approximately $180 this month. Would you like to see which categories are contributing most?"

**Bad — Urgent, Vague, No Reasoning:**
> "⚠️ ALERT: You're overspending! Click here to fix it."

**Good — Transparent, Honest, Supports Decision:**
> "Your portfolio is down 2.3% this week. This is within normal market fluctuation for your allocation. No action is needed unless your circumstances have changed. Would you like to review your allocation?"

**Bad — Alarming, Hides Reasoning:**
> "Your portfolio is losing money! Consider selling now."

**Good — Educational, Non-Judgmental:**
> "You have three subscriptions totaling $47/month that you haven't used in 90+ days. Would you like to review them?"

**Bad — Judgmental, Pressure:**
> "You're wasting money on unused subscriptions. Cancel now to save."

### Language Patterns

| Pattern | Example |
|---|---|
| **Recommend, don't command** | "You might consider..." / "One option is..." / "Based on your goals..." |
| **Explain, don't assume** | "This is based on..." / "The reasoning is..." / "The data shows..." |
| **Offer, don't push** | "Would you like to..." / "If you're interested..." / "I can show you..." |
| **Acknowledge uncertainty** | "I'm not certain about..." / "The data suggests... but..." / "This depends on..." |
| **Invite, don't demand** | "Would you like me to..." / "Want to explore..." / "Should I look into..." |

## 04 — AI Entry Points

### Philosophy

Never require users to navigate to a separate AI page for common tasks. AI appears contextually where decisions are made.

### Entry Point Locations

| Location | AI Entry Point | Trigger |
|---|---|---|
| **Dashboard** | Hero insight, proactive alerts | Daily summary, anomalies, opportunities |
| **Portfolio** | Allocation suggestions, rebalancing | Market changes, drift from targets |
| **Goals** | Progress insights, adjustment suggestions | Milestones, shortfalls, timeline changes |
| **Bills** | Payment reminders, savings opportunities | Upcoming due dates, duplicate charges |
| **Automation** | Suggestion to automate, monitoring | Recurring patterns detected |
| **Transactions** | Categorization, anomaly detection | Unusual charges, pattern changes |
| **Investments** | Research insights, risk analysis | Market events, portfolio impact |
| **Settings** | Preference learning, personalization | Usage patterns |
| **Search** | AI-enhanced answers | Complex financial queries |
| **Notifications** | Contextual recommendations | Triggered by events |

### Entry Point Patterns

**Proactive — AI Appears Without Being Asked:**
```
┌──────────────────────────────────────────────────┐
│  ✦ Insight                                       │
│                                                  │
│  You have $2,400 in checking that could be       │
│  earning 4.5% APY in your savings account.       │
│                                                  │
│  [Move funds] [Tell me more] [Dismiss]           │
└──────────────────────────────────────────────────┘
```

**Reactive — AI Appears When User Invokes:**
```
┌──────────────────────────────────────────────────┐
│  [Ask AI about this transaction]                 │
│                                                  │
│  User clicks → AI explains context,              │
│  categorization, or provides related insights    │
└──────────────────────────────────────────────────┘
```

**Contextual — AI Appears Based on Screen Content:**
```
┌──────────────────────────────────────────────────┐
│  Your portfolio is down 3.2% this month.         │
│                                                  │
│  ✦ Context: This is normal volatility for your   │
│  risk profile. Your 1-year return is +8.7%.     │
│  [View analysis]                                 │
└──────────────────────────────────────────────────┘
```

## 06 — AI Recommendation Cards

### Anatomy of Every Recommendation

Every recommendation card MUST include:

| Element | Description |
|---|---|
| **Recommendation** | Clear, specific action to take |
| **Reason** | Why this recommendation exists (data-backed) |
| **Expected Impact** | What the user gains (quantified when possible) |
| **Confidence Level** | How certain the AI is (High/Medium/Low) |
| **Risks** | What could go wrong or unexpected outcomes |
| **Alternative Options** | Other paths the user could consider |
| **Primary Action** | One clear CTA to execute the recommendation |
| **Dismiss Option** | Easy way to say "not now" without pressure |
| **Learn More** | Path to deeper explanation |

### Recommendation Structure

```
┌──────────────────────────────────────────────────┐
│  ✦ Recommended: [Action]                         │
│──────────────────────────────────────────────────│
│                                                  │
│  REASON:                                         │
│  [Why this recommendation exists, grounded in    │
│  the user's data and financial situation]        │
│                                                  │
│  EXPECTED IMPACT:                                │
│  [Quantified benefit — dollar amount, percentage,│
│  time saved, risk reduced]                       │
│                                                  │
│  CONFIDENCE: [High/Medium/Low]                   │
│  [Plain language explanation of confidence]      │
│                                                  │
│  RISKS:                                          │
│  • [Risk 1]                                      │
│  • [Risk 2]                                      │
│                                                  │
│  ALTERNATIVES:                                   │
│  • [Option B] — [Trade-off]                      │
│  • [Option C] — [Trade-off]                      │
│                                                  │
│  ─────────────────────────────────────────────── │
│                                                  │
│  [Take action]  [Not now]  [Why this?]           │
└──────────────────────────────────────────────────┘
```

### Rules

1. Never present recommendations without context
2. Always explain the "why" before the "what"
3. Quantify impact whenever possible
4. Present alternatives — never imply only one option exists
5. Confidence must be explained, never just displayed
6. Risks must be honest and complete
7. "Not now" is always available — never only "Dismiss"
8. "Why this?" always leads to full reasoning breakdown

## 07 — AI Conversation Patterns

### Conversation Start

**Context-Aware Opening:**
```
┌──────────────────────────────────────────────────┐
│  ✦ What would you like to know?                  │
│                                                  │
│  Based on your recent activity, you might want   │
│  to know:                                        │
│                                                  │
│  • Why your spending increased this month        │
│  • Whether you're on track for your goal         │
│  • How your portfolio performed today            │
│                                                  │
│  [Ask something else]                            │
└──────────────────────────────────────────────────┘
```

### Context-Aware Suggestions

Suggestions appear based on what the user is currently viewing:

**On Portfolio Screen:**
- "Why is my portfolio down today?"
- "How does my allocation compare to my risk profile?"
- "What's driving my returns this quarter?"

**On Bills Screen:**
- "Which bills can I automate?"
- "Am I overpaying for any subscriptions?"
- "When is my next payment due?"

**On Goals Screen:**
- "Am I on track for my emergency fund goal?"
- "What if I increase my monthly contribution by $200?"
- "When will I reach my goal at current pace?"

### Follow-up Questions

After an AI response, suggest relevant follow-ups:
```
┌──────────────────────────────────────────────────┐
│  ✦ Follow up                                     │
│                                                  │
│  • "Can you break that down by category?"        │
│  • "What's the trend over the last 6 months?"   │
│  • "How does this compare to my budget?"         │
│  • "What should I do about it?"                  │
└──────────────────────────────────────────────────┘
```

### Clarification

When the user's question is ambiguous:
```
┌──────────────────────────────────────────────────┐
│  ✦ Let me clarify                                │
│                                                  │
│  When you say "my portfolio," are you asking     │

## 08 — AI Reasoning

### The Right to Explanation

Every AI recommendation must explain itself. Users have the right to understand why something was suggested.

### Reasoning Structure

Every recommendation explains:

| Element | Description |
|---|---|
| **What happened** | The factual basis for the recommendation |
| **Why it matters** | The impact on the user's financial situation |
| **How it was calculated** | The methodology and data used |
| **Available alternatives** | Other options the user could consider |
| **Potential risks** | What could go wrong |
| **Assumptions** | What the AI is assuming to be true |

### Reasoning Display

**Inline Reasoning:**
```
Your dining spending increased 23% over 3 months.
This matters because it could exceed your monthly budget by ~$180.

How this was calculated:
• Analyzed 847 transactions (Sep-Nov 2025)
• Compared to your stated budget of $600/month
• Projected forward using current trend
```

**Detailed Reasoning Panel:**

│──────────────────────────────────────────────────│

## 09 — AI Confidence

### Confidence System

Every AI recommendation includes a confidence level that reflects the certainty of the analysis.

### Confidence Levels

| Level | Label | Plain Language | Usage |
|---|---|---|---|
| **High** | "Very confident" | Strong data, clear pattern, consistent history | 3+ months of consistent data, no anomalies |
| **Medium** | "Somewhat confident" | Good data but some uncertainty | 1-3 months of data, minor anomalies |
| **Low** | "Limited data" | Insufficient data for strong conclusion | Less than 1 month, incomplete data |
| **Unknown** | "Insufficient data" | Cannot determine confidence | New user, minimal transactions |

### Confidence Display

**High Confidence:**
```
Confidence: High ✓
"Very confident — based on 3 months of consistent
spending data with no unusual events."
```

**Medium Confidence:**
```
Confidence: Moderate ⚠
"Somewhat confident — data is limited to 6 weeks.
Recommendation may change as more data becomes available."
```

**Low Confidence:**
```
Confidence: Low ⚠
"Limited data — this analysis is based on only 12
transactions. Take this as a starting point, not a
final recommendation."
```

**Unknown Confidence:**
```
Confidence: Unable to determine ⚠
"Not enough data to assess confidence. This recommendation
is based on general best practices, not your specific situation."
```

### Confidence Explanation

Confidence is always explained using plain language, never just a percentage:

**Never:**
```
Confidence: 73%
```

**Always:**
```
Confidence: Medium (73%)
"Somewhat confident — based on 6 weeks of data.
Your spending patterns are becoming clear, but a full
month of data would improve accuracy."
```

### Confidence Factors

Display what contributes to confidence:

```
What contributes to this confidence level:
✓ 3 months of transaction data
✓ Clear spending pattern identified
✓ No unusual one-time events
✓ Consistent category boundaries

What could improve confidence:
• More historical data
• Category refinement
• Budget goal confirmation
```

   - Alternative options

3. **Undo window:**

   - Critical actions have a 5-minute undo window

## 16 — AI Ethics

### Ethical Principles

| Principle | Description |
|---|---|
| **Transparency** | AI always identifies itself and its limitations |
| **User Consent** | Data usage is explicit and opt-in |
| **Privacy** | User data is protected and never shared without consent |
| **Data Usage** | Data is used only for stated purposes |
| **Recommendation Limitations** | AI never claims to be financial advice |
| **Bias Awareness** | AI acknowledges potential biases in analysis |
| **No Manipulation** | AI never uses dark patterns or pressure tactics |
| **No Hidden Objectives** | AI's purpose is always clear |
| **No Guaranteed Outcomes** | AI never promises financial results |

### Transparency Requirements

1. **AI Identification:** Every AI-generated content is clearly marked
2. **Data Sources:** Users know what data informs recommendations
3. **Methodology:** Analysis methods are explainable
4. **Limitations:** AI clearly states what it cannot do
5. **Auditability:** Every AI decision has a reasoning trail

### Privacy Protections

1. **Data Minimization:** Only collect data needed for stated purposes
2. **Purpose Limitation:** Data used only for what user authorized
3. **Storage Limitation:** Data retained only as long as needed
4. **User Control:** Users can access, modify, and delete their data
5. **No Sharing:** Data never shared with third parties without explicit consent

### Consent Framework

```
┌──────────────────────────────────────────────────┐

- Methodology applied
- Confidence calculated
- Recommendation generated
- User action taken (if any)

### Rules

1. Every AI decision should be auditable
2. Users can request full audit of their AI interactions
3. AI cannot delete audit records
4. Audit data retained for regulatory compliance
5. Users can export their audit data
